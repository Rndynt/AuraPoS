# Conflict Resolution Guidelines

Initial conflict classes:
- PRODUCT_INACTIVE
- PRICE_CHANGED
- STOCK_INSUFFICIENT
- ORDER_DUPLICATE
- PAYMENT_DUPLICATE
- TENANT_FEATURE_DISABLED
- TERMINAL_INACTIVE

Severity model:
- warning
- needs_review
- blocking

Sprint 1 introduces conflict storage primitives only; resolver policies are implemented in later sprints.
